RUBY ON RAILS Ruby Installeren: !!!ELKE REGEL MOET STUK VOOR STUK GEPLAATST WORDEN!!! 1. Open Terminal 2. We gaan nu alle benodigdheden voor Webpacker support in Rails toevoegen, eerst voegen we Node.js & Yarn repositories in ons systeem voor dat we ze installeren: curl -sL https://deb.nodesource.com/setup_8.x | sudo -E bash - curl -sS https://dl.yarnpkg.com/debian/pubkey.gpg | sudo apt-key add - echo "deb https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list sudo apt-get update sudo apt-get install git-core curl zlib1g-dev build-essential libssl-dev libreadline-dev libyaml-dev libsqlite3-dev sqlite3 libxml2-dev libxslt1-dev libcurl4-openssl-dev software-properties-common libffi-dev nodejs yarn 3. Als volgt gaan we Ruby installeren met de ‘rebnv’ methode: Installeren met rbenv is een simpele twee stappen process. Als eerst installeer je rbenv, en dan ruby: Cd git clone https://github.com/rbenv/rbenv.git ~/.rbenv Sudo nano ~/.rbenv Voeg de volgende regels onderin toe: export PATH="$HOME/.rbenv/bin:$PATH" eval "$(rbenv init -)" export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH" Sluit je bashrc file met control+o, enter en dan control+x exec $SHELL git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build exec $SHELL rbenv install 2.5.1 rbenv global 2.5.1 ruby -v 4. De laatste stap is Bundler installeren: gem install bundler Git Configureren: 1. We gebruiken Git voor onze version control system, dus we stellen het in om overeen te komen met onze GitHub account: Verander “YOUR NAME” in je GitHub gebruikersnaam Verander “YOUR@EMAIL.COM” in je GitHub email git config --global color.ui true git config --global user.name "YOUR NAME" git config --global user.email "YOUR@EMAIL.com" ssh-keygen -t rsa -b 4096 -C "YOUR@EMAIL.com" 2. Nu moet de zojuist gegenereerde SSH token toegevoegd worden in het dashboard van Github zodat je jouw computer autoriseert om commits te maken: cat ~/.ssh/id_rsa.pub ssh -T git@github.com 3. Als het goed is moet je zo’n bericht krijgen: Hi excid3! You've successfully authenticated, but GitHub does not provide shell access. Rails Installeren: 1. Om Node.JS te installeren, gaan we het toevoegen met behulp van de officiële repository: curl -sL https://deb.nodesource.com/setup_8.x | sudo -E bash - sudo apt-get install -y nodejs 2. Als volgt: gem install rails -v 5.2.0 3. Met ‘rbenv’ moet je de volgende code uitvoeren om het uitvoerbare bestand van de rails beschikbaar te maken: rbenv rehash 4. Nu dat je Rails hebt geïnstalleerd, kan je de ‘rails-v’ command runnen om zeker te weten of je alles goed hebt geïnstalleerd: rails -v # Rails 5.2.0 PostgreSQL: 1. Voor PostgreSQL, gaan we een nieuwe repository toevoegen voor de meest recente versie van Postgres: sudo sh -c "echo 'deb http://apt.postgresql.org/pub/repos/apt/ xenial-pgdg main' > /etc/apt/sources.list.d/pgdg.list" wget --quiet -O - http://apt.postgresql.org/pub/repos/apt/ACCC4CF8.asc | sudo apt-key add - sudo apt-get update sudo apt-get install postgresql-common sudo apt-get install postgresql-9.5 libpq-dev 2. De Postgres installatie maakt geen user voor jou, dus zijn er nog 3 stappen om een user te maken met toestemming om een databases te maken. 3. Verander “chris” met jouw gebruikersnaam: sudo -u postgres createuser chris -s # If you would like to set a password for the user, you can do the following sudo -u postgres psql postgres=# \password chris Final steps: 1. Eindelijk je eerste Rails applicatie! #### If you want to use SQLite (not recommended) rails new database #### If you want to use Postgres # Note that this will expect a postgres user with the same username # as your app, you may need to edit config/database.yml to match the # user you created earlier rails new database -d postgresql # Move into the application directory cd super_megan_tutorial # If you setup MySQL or Postgres with a username/password, modify the # config/database.yml file to contain the username/password that you specified # Create the database rails db:create rails s You can now visit http://localhost:3000 to view your new website! If you received an error that said Access denied for user 'root'@'localhost' (using password: NO) then you need to update your config/database.yml file to match the database username and password.
